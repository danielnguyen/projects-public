# Agent Workflow

This directory contains public-facing rules for AI-assisted implementation work.

The central premise is simple:

> Specifications are authoritative. Agent memory is not.

For long-running AI-assisted projects, this matters because coding agents can drift, overfit to prior conversation, invent missing requirements, or silently rewrite implementation scope.

The workflow rules here are intended to keep AI-assisted implementation grounded in written specifications, current code, explicit ownership boundaries, and verifiable validation.

## Included files

- `AGENTS.md` — operating rules for AI coding agents
- `ENGINEERING_RULES_excerpt.md` — naming and runtime cleanliness rules
