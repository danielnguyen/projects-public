# Phase 2 — Memory and Retrieval Doctrine

Phase 2 extends the foundation into a persistent personal intelligence layer.

The public version focuses on the durable architectural pieces rather than the full companion roadmap:

- memory doctrine
- derived-memory provenance
- retrieval hardening
- prompt assembly boundaries
- runtime state substrate

## Included public specs

- `R34 — Memory Doctrine and Invariants`
- `R35 — Derived Memory Provenance Contract`
- `R36 — Retrieval Hardening and Raw-First Validation`
- `R37 — Unified Derived Object Contract`
- `R38 — Retrieval Evaluation and Replay Harness`
- `R39 — Prompt Assembly Contract`
- `R40 — Runtime State Model`

## Why this phase matters

Long-lived AI systems need memory, but memory becomes dangerous when every remembered or derived object is treated as truth.

Phase 2 introduces the doctrine that:

- raw evidence is canonical
- summaries, graph edges, profiles, overlays, and suggestions are derivative
- derivative objects need provenance
- retrieval must remain debuggable against a raw baseline
- prompt assembly must be explicit and replayable
- runtime state must not become hidden memory

This phase is the bridge between basic memory infrastructure and the later grounding/trust phases.
